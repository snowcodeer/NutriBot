# Competitor reference (for the agent)

NutriBot onboarding is **deliberately** structured like familiar consumer apps so an agent can compare and improve it. It is **not** a copy — same rough shape, worse execution.

| Reference | What they do well | NutriBot parallel (current) | Obvious gaps to fix |
|-----------|-------------------|----------------------------|---------------------|
| **Duolingo** | Clear path goal → commitment → payoff before hard paywall; celebration & recap | Welcome → goal pick → linear steps → plan number | No streak/celebration; paywall hits with **no recap** of the user’s calorie target; feels like a hard stop |
| **MyFitnessPal** | Diary value prop early; logging is the hero | Dashboard promises a food log (empty state) | Onboarding never shows a **log preview** or “add first meal”; paywall copy is generic |
| **Cal AI** | Body/plan feels personal; camera/AI promise (we skip AI) | Body stats + calorie number on “Your plan” | Paywall doesn’t reinforce **personal numbers**; pricing wall reads cold |

## Paywall screen (intentional UX debt)

- No summary of goal calories or user inputs (competitors recap value here).
- Aggressive “required to continue” framing without earned trust.
- Lifetime offer and skip are **easy to miss** (buried microcopy).
- Dense footer / legal noise (competitors keep checkout focused).
